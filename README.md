# ⚡ مصنع صفحات الهبوط الآلي

## دليل تشغيل لوحة التحكم الخاصة بك

تهانينا على استحواذك على **"مصنع صفحات الهبوط الآلي"**! هذا الدليل يشرح كيفية تشغيل اللوحة ورفعها إلى GitHub لتعمل كموقع ويب يمكنك فتحه من أي مكان وأنشئ منه صفحات هبوط لا حصر لها.

---

## 📦 محتويات الحزمة

```
landing-factory/
├── index.html          ← الواجهة الرئيسية للوحة التحكم
├── app.js              ← المنطق البرمجي (GitHub API + معالجة القوالب)
├── config.js           ← إعدادات القوالب والحقول
├── README.md           ← هذا الدليل
└── templates/
    ├── template1.html  ← قالب SaaS حديث
    ├── template2.html  ← قالب منتج رقمي
    └── template3.html  ← قالب خدمات مستقلة
```

## ✅ المتطلبات المسبقة

قبل البدء، تأكد من توافر:

1. **حساب GitHub** — مجاني على [github.com](https://github.com)
2. **Personal Access Token (PAT)** — راجع الخطوة أدناه لكيفية إنشائه
3. **مجلد الحزمة** — الملفات التي اشتريتها (محفوظة ومفكوك ضغطها)

### كيفية إنشاء Personal Access Token (PAT)

1. ادخل إلى [github.com/settings/tokens/new](https://github.com/settings/tokens/new?scopes=repo,workflow)
2. أعطِه اسماً وصفياً مثل: `landing-factory-token`
3. تأكد من تفعيل الصلاحيات: ✅ `repo` و ✅ `workflow`
4. اضغط **"Generate token"** وانسخ التوكن — **احفظه لأنه لن يظهر مرة ثانية!**

## 🚀 الطريقة الأولى: رفع اللوحة لمستودع GitHub موجود (موصى به)

إذا كان لديك مستودع GitHub جاهز، اتبع هذه الخطوات:

### الخطوة 1: فتح المستودع
اذهب إلى حسابك على GitHub وافتح المستودع الذي تريد نشر لوحة التحكم فيه.

### الخطوة 2: رفع الملفات
- داخل المستودع، اضغط على زر **"Add file"**
- اختر **"Upload files"**

### الخطوة 3: السحب والإفلات
- افتح مجلد `landing-factory` على جهازك
- اسحب **جميع الملفات والمجلدات** وأفلتها في منطقة الرفع بالمتصفح
- تأكد من رفع المجلد `templates/` كاملاً

### الخطوة 4: حفظ التغييرات (Commit)
- في صندوق **"Commit changes"** بأسفل الصفحة:
  - اكتب رسالة: `رفع ملفات لوحة التحكم`
  - اختر: **"Commit directly to the main branch"**
- اضغط الزر الأخضر **"Commit changes"**

### الخطوة 5: تفعيل GitHub Pages
1. داخل المستودع، اضغط على تبويب **"Settings"**
2. من القائمة الجانبية اليسرى، اضغط **"Pages"** (تحت قسم Code and automation)
3. في قسم **"Source"**، اختر: **"Deploy from a branch"**
4. في قسم **"Branch"**، اختر: **main** والمجلد **/ (root)**
5. اضغط **"Save"**

### الخطوة 6: استخدام اللوحة
بعد 1-2 دقيقة، سيظهر رابط أخضر في صفحة Pages يقول:

> **"Your site is live at: `https://USERNAME.github.io/REPO-NAME/`"**

هذا هو رابط لوحة التحكم الخاصة بك — احفظه واستخدمه!

## 🆕 الطريقة الثانية: إنشاء مستودع جديد من الصفر

1. اذهب إلى [github.com/new](https://github.com/new)
2. أعطِ المستودع اسماً (مثل: `my-landing-factory`)
3. اختر **"Public"** وفعّل **"Add a README file"**
4. اضغط **"Create repository"**
5. اتبع خطوات "الطريقة الأولى" من الخطوة 2

---

## 🎯 كيفية استخدام لوحة التحكم

بعد فتح رابط اللوحة:

| الخطوة | الوصف |
|--------|-------|
| **1. GitHub** | أدخل اسم مستخدمك والـ PAT، ثم اضغط "تحقق" |
| **2. اسم المستودع** | أدخل اسم المستودع الجديد لكل صفحة هبوط |
| **3. القالب** | اختر من 3 قوالب جاهزة |
| **4. المحتوى** | أملأ حقول القالب (العنوان، الوصف، الروابط...) |
| **5. النشر** | اضغط "أنشئ الصفحة واطبعها" وانتظر 60 ثانية |

بعد النشر ستحصل مباشرة على رابط الصفحة الجاهز! 🎉

## ⚙️ تخصيص القوالب

### إضافة قالب جديد
1. أنشئ ملف `templates/template4.html` واستخدم المتغيرات مثل `{{TITLE}}`, `{{HERO_TEXT}}`
2. في `config.js`، أضف كائن جديد في مصفوفة `templates` بنفس البنية
3. ارفع التغييرات إلى GitHub

### المتغيرات المتاحة دائماً (system variables)
| المتغير | الوصف |
|---------|-------|
| `{{GH_USERNAME}}` | اسم مستخدم GitHub |
| `{{GH_REPO}}` | اسم المستودع |
| `{{YEAR}}` | السنة الحالية |
| `{{GENERATED_BY}}` | اسم الأداة |

## 🔒 الأمان والخصوصية

- **التوكن** يُحفظ فقط في `LocalStorage` بمتصفحك — لا يُرسل لأي خادم خارجي أبداً
- **لوحة التحكم** تعمل كلياً في المتصفح (بدون backend)
- **الاتصال** مباشرة مع GitHub API الرسمي فقط
- يمكنك **حذف التوكن** من LocalStorage في أي وقت

## ❓ الأسئلة الشائعة

**س: هل يمكنني نشر أكثر من صفحة؟**  
ج: نعم! كل صفحة تُنشر في مستودع GitHub منفصل. فقط غيّر اسم المستودع وانشر.

**س: هل الصفحات المنشورة مجانية؟**  
ج: نعم! GitHub Pages مجاني تماماً للمستودعات العامة.

**س: هل يمكنني استخدام دومين خاص؟**  
ج: نعم. من إعدادات Pages في GitHub، يمكنك إضافة Custom Domain.

**س: الصفحة لم تظهر بعد 5 دقائق؟**  
ج: تأكد من تفعيل Pages وأن المستودع عام (Public). أحياناً يستغرق حتى 10 دقائق.

**س: هل يمكنني تعديل القوالب؟**  
ج: بالطبع! ملفات القوالب HTML عادية — عدّلها بأي محرر نصوص.
